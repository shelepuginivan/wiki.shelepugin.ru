На этой странице описаны все верхнеуровневые опции, которые не имеют отдельных
страниц.

Пример, включающий все эти опции:

```kdl
spawn-at-startup "waybar"
spawn-at-startup "alacritty"

prefer-no-csd

screenshot-path "~/Pictures/Screenshots/Screenshot from %Y-%m-%d %H-%M-%S.png"

environment {
    QT_QPA_PLATFORM "wayland"
    DISPLAY null
}

cursor {
    xcursor-theme "breeze_cursors"
    xcursor-size 48
}

hotkey-overlay {
    skip-at-startup
}
```

## `spawn-at-startup`

Добавьте, чтобы запустить процессы при запуске niri.

`spawn-at-startup` принимает путь до исполняемого файла, за которым следуют аргументы.

Эта опция работает точно также, как действие
[комбинаций клавиш `spawn`](/niri/configuration/binds#spawn),
поэтому, пожалуйста, ознакомьтесь с нюансами на соответствующей странице.

```kdl
spawn-at-startup "waybar"
spawn-at-startup "alacritty"
```

Обратите внимание, что запуск niri в качестве сессии systemd поддерживает
xdg-desktop-autostart из коробки, что может быть удобнее.
Благодаря этому приложения, которые вы настроили на автозапуск в GNOME,
будут "просто работать" и в niri, без какой-либо ручной настройки `spawn-at-startup`.

## `prefer-no-csd`

Этот флаг заставит niri указать приложениям, что они не должны использовать
декорации на стороне клиента.

Если приложение отдельно "попросит" использовать их, то запрос будет проигнорирован.
Кроме того, приложениям будет дана информация, что они выложены плиткой (tiled),
что уберёт некоторые скруглённые углы.

Если установлено значение prefer-no-csd, приложения, которые согласовывают
декорации на стороне сервера через протокол xdg-decoration, будут иметь кольцо
фокуса и границы, нарисованные вокруг них _без_ сплошного цветного фона.

:::note

В отличие от большей части других опций, изменение `prefer-no-csd` не повлияет
на уже запущенные приложения. Это связано, по большей части, с тем, что niri
обходит [баг в SDL2](https://github.com/libsdl-org/SDL/issues/8173), который
предотвращает запуск SDL2-приложений.

Перезапустите приложения после изменения `prefer-no-csd`, чтобы применить новую
конфигурацию.

:::

```kdl
prefer-no-csd
```

## `screenshot-path`

Устанавливает путь, куда сохраняются снимки экрана. Тильда `~` в самом начале
будет заменена на путь до домашней директории.

Путь форматируется с помощью
[`strftime(3)`](https://man.archlinux.org/man/strftime.3),
чтобы предоставить вам дату и время снимка экрана.

Niri создаст последнюю директорию пути, если та не существует.

```kdl
screenshot-path "~/Pictures/Screenshots/Screenshot from %Y-%m-%d %H-%M-%S.png"
```

Вы также можете указать значение `null` для этой опции, чтобы отключить
сохранение снимков экрана на диске:

```kdl "null"
screenshot-path null
```

## `environment`

Переопределяет переменные окружения для процессов, запускаемых niri.

```
environment {
    // Установить переменную следующим образом:
    // QT_QPA_PLATFORM "wayland"

    // Удалить переменную, указав значение null:
    // DISPLAY null
}
```

## `cursor`

Изменить тему и размер курсора, а также установить переменные окружения
`XCURSOR_THEME` и `XCURSOR_SIZE`.

```kdl
cursor {
    xcursor-theme "breeze_cursors"
    xcursor-size 48
}
```

## `hotkey-overlay`

Настройка для подсказки "Important Hotkeys".

Установите флаг `skip-at-startup`, если вы не хотите видеть эту подсказку при
запуске niri.


```kdl
hotkey-overlay {
    skip-at-startup
}
```
