Вы можете объявить именованные рабочие пространства на верхнем уровне конфигурации:

```kdl
workspace "browser"

workspace "chat" {
    open-on-output "DP-2"
}
```

В отличие от обычных динамических рабочих пространств, именованные существуют
всегда, даже если на них нет окон. В противном случае они работают точно также,
как и обычные рабочие пространства: вы можете двигать их, перемещать на другие
мониторы и т.д.

Действия, такие как `focus-workspace` или `move-column-to-workspace` могут
ссылаться на рабочие пространства по имени. А также, вы можете использовать
[правило](/niri/configuration/window-rules) `open-on-workspace`, чтобы
заставить окна открываться на указанном именованном рабочем пронстранстве:

```
// Задать рабочее пространство "chat", которое открывается на
// устройстве вывода "DP-2"
workspace "chat" {
    open-on-output "DP-2"
}

// Открывать Fractal на рабочем пространстве "chat", если он запускается при
// запуске niri
window-rule {
    match at-startup=true app-id=r#"^org\.gnome\.Fractal$"#
    open-on-workspace "chat"
}
```

Именованные рабочие пространства изначально отображаются в том порядке, в
котором они объявлены в файле конфигурации. При редактировании конфигурации во
время работы niri вновь объявленные именованные рабочие пространства будут
появляться в самом верху монитора.

Если вы удалите из конфигурации какое-либо именованное рабочее пространство,
оно станет обычным (неименованным), и если на нем не будет окон, оно будет
удалено (как и любое другое обычное рабочее пространство). Нет способа
присвоить имя уже существующему рабочему пространству, но вы можете можно
просто переместить нужные окна в новое именованное пространство.
